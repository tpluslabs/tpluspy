# Contributing to tpluspy

## Development setup

```shell
uv pip install -e ".[test,lint]"
```

## Linting and tests

```shell
ruff check tplus tests
ruff format --check tplus tests
mypy tplus
pytest
```

## Building the docs

The documentation lives in [`docs/`](./docs) and is built with
[Sphinx](https://www.sphinx-doc.org/) using the
[MyST](https://myst-parser.readthedocs.io/) markdown parser. Method
references are generated by `sphinx.ext.autodoc` from Google-style
docstrings.

Install the `docs` extra to pull in Sphinx, the MyST parser, the Read the
Docs theme, and the live-reload server:

```shell
uv pip install -e ".[docs]"
```

Then build the HTML site:

```shell
cd docs
make html
```

The generated site lands in `docs/_build/html`. Open the entry point in a
browser:

```shell
open docs/_build/html/index.html       # macOS
xdg-open docs/_build/html/index.html   # Linux
```

To remove generated artefacts:

```shell
cd docs && make clean
```

### Live preview

For a live-reloading preview while editing, run:

```shell
cd docs
make livehtml
```

This starts `sphinx-autobuild` (installed by the `docs` extra), serves the
site at <http://127.0.0.1:8000>, and rebuilds whenever a source file
changes.

### Adding pages

- User guides go under `docs/userguides/` as MyST markdown (`.md`). Add the
  new file to the `userguides` toctree in `docs/index.md`.
- API reference pages go under `docs/methoddocs/` and use
  `automodule`/`autoclass` directives inside ` `{eval-rst} \`\`\` fences.
- New documented modules pick up automatically from their Google-style
  docstrings -- no extra wiring required beyond an `autoclass` /
  `automodule` entry.
